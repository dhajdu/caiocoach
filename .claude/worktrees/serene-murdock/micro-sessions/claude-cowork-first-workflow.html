Session
03 Concept

The Problem

Think about your last five days of work. How much of it was creating something genuinely new - and how much was moving information from one place to another?

Summarizing a meeting. Formatting a status report. Sorting downloaded files. Pulling key numbers out of a document. These tasks are real. They take real time. And almost none of them require a human. They require a system.

Without a workflow

You read through your notes manually. You retype key information into a shared doc. You spend 45 minutes on a task that is identical every single week.

With a workflow

Claude reads all your notes at once. It builds the summary. You review and send. The same output in five minutes - and someone else on your team can run it too.

Key Insight

Half your recurring work is file work. That half is now automatable. An AI Officer's job is to identify which half and build the systems to move it.

04 How It Works

What Cowork Does

Claude Cowork is a desktop application. You give it access to a folder on your computer. You write a plain-language instruction. Claude reads the files, does the work, and saves the output.

No API keys. No code. No setup beyond a folder and a sentence. The instruction is the workflow.

What you can build right now

1

Weekly document briefing

Summarize a folder of documents into a single short briefing. One paragraph per file, one executive summary across all files.

2

File organization

Sort a messy folder by topic, date, or project. Get an indexed list of what's in there and where each file belongs.

3

Action item extractor

Pull every task, deadline, and next step from a set of meeting notes or project documents into a single table.

4

Friday status update

Draft a weekly status report from raw notes - what was completed, what is in progress, what is blocked, what comes next.

Writing a strong instruction

Be specific about three things: the input (what files and what they contain), the process (what to do with them), and the output (what to produce and how to save it). Vague instructions produce vague results. Specific instructions produce consistent results every time.

05 Exercise

Exercise: Write Your First Instruction

Everyone does this now. Take two minutes.

Think of one recurring task you do with files - documents, reports, notes, anything. Now write a one-sentence instruction you would give to a smart new hire. Not "do my work" - something specific.

Weak instruction

"Summarize my files."

Strong instruction

"Read all the files in this folder and write a one-paragraph summary of each document, then write a single-paragraph executive summary of the key themes. Save the output as Weekly-Briefing.md."

Write your instruction. Post it in the chat. Your trainer will show you what Cowork does with it.

What to notice

The best instructions are ones you would give to a capable person on their first day. They assume intelligence but not context. They define the output, not just the task.

06 Leadership

The Leadership Layer

Here is the shift. When you set up a Cowork workflow, you are not just saving yourself 20 minutes. You are building something that someone else can run.

A new team member. A colleague covering for you. Your manager who needs the same briefing every Friday.

The AI Officer Test

An AI Officer does not just use AI tools - they build AI systems that work without them. Ask this question about every workflow you build: could someone else run this and get consistent results without asking me anything?

This is what that question looks like in practice. You build it for yourself first. You make it work. Then you hand it to your team. That is the progression from individual contributor to AI Officer.

Using AI

You run a workflow. You save 20 minutes. Only you know how to run it. If you leave, the workflow leaves with you.

Leading AI

You build a workflow. You document the instruction. You hand it to your team. The system runs whether you are in the room or not.

08 Recap

Key Takeaways

  • Half your recurring work is file work Moving, summarizing, formatting, organizing - these tasks are real and they are now automatable. That is where you start.
  • Cowork runs on plain-language instructions No code required. Connect a folder. Write what you want done. Run it. The instruction is the workflow.
  • Specific instructions produce consistent results Define the input, the process, and the output. Vague instructions produce vague results. Treat it like briefing a capable new hire.
  • The task is not the point - the system is When you build a workflow, you are building something someone else can run. That is the difference between using AI and leading it.
  • Build it for yourself first, then hand it to your team Every system starts as a personal tool. An AI Officer's job is to make it transferable. That is the leadership test.
09 Action

Try This Week

Pick the workflow instruction you wrote during the session. Open Cowork. Connect a real folder from your work. Run it.

If the output is not perfect the first time, adjust the instruction and run it again. The first version is always a draft. The second version is usually close. The third version is the one you keep.

Once it works, share it with one person on your team. Show them the instruction and the output. Ask if they have a task it could run on. That is how a personal workflow becomes a team system.
