# CAIO Coach — Resources

Brand-level resource files for the CAIO Coach project. These guide how work gets done across content, publishing, and strategy. Read by humans and Claude — not queried by apps.

## Files

| File | What it is | When to reference |
|---|---|---|
| **content-calendar.md** | What to publish, when, which channels | Before starting any content production |
| **seo-strategy.md** | Keywords, target pages, search + AI search goals | During SEO step in /studio, when building pages |
| **brand-voice.md** | Dave's writing voice, worldview, what to avoid | Before writing anything for CAIO Coach |
| **design-system.md** | Visual design — colors, typography, components, HTML patterns | When building pages or circle.html |
| **audience-personas.md** | (planned) Canonical audience personas | When targeting content |
| **pricing-strategy.md** | (planned) Programs, pricing, positioning | When creating sales or conversion content |

## Rules

- These files are the canonical source of truth for CAIO Coach brand decisions
- The content studio's `channels-and-guidelines.md` should reference these when it needs deeper detail
- Update these files when strategy changes — they inform every piece of content and every page built
- If a resource file doesn't exist yet, create it here when the need arises

## Database vs Resource Files

**Put it in Supabase if:** it changes frequently, gets queried by an app, has relationships, or comes from form submissions (contacts, enrollments, payments, analytics).

**Put it here if:** it guides how work gets done, changes occasionally, and is read by humans or Claude (strategy, calendars, style guides, personas, checklists).
