03 Concept

A Prompt Is Not a Brief

Most people write prompts. Very few people write briefs. They are not the same thing.

A prompt

A one-time instruction you give to get a one-time result. "Summarize this document." "Write a LinkedIn post about X." Works for conversational AI. Does not work for agents.

A brief

A permanent set of instructions that governs how an agent behaves across every request it handles. Think of it as a job description. You write it once. It runs every time.

When you run an agent on a brief, it will follow those instructions literally - for every request, indefinitely, without asking for clarification. That is why the brief has to be complete before you deploy it.

Key Insight

An agent brief is a job description for someone who never asks for clarification. If you would not give this brief to a new hire on their first day without a follow-up conversation, it is not ready.

04 Framework

The 4 Components of a Strong Brief

A production-ready agent brief has four things. Most people only write two of them.

1

Context

Who is this agent? What is its role? What does it know about the organization, the audience, and the purpose of its work? Context shapes everything else.

2

Task

What should the agent do with what input? Be specific. "Summarize documents" is not a task. "Read the document and write a three-paragraph summary with the key decision on the first line" is a task.

3

Constraints

What should the agent never do? What tone, format, or content is off-limits? Constraints prevent the most common failure modes before they happen.

4

Output format

Exactly what should the result look like? Headers, sections, length, file type. If the output format is vague, the agent invents one - and it will be different every time.

The most skipped component

Output format. Everyone writes context and task. Almost nobody defines the output format with enough precision. This is the most common reason agents produce inconsistent results.

05 Exercise

Write a Brief Live

Everyone does this now. Five minutes.

Do this right now

Think of an agent you have built or want to build. Write a brief for it using the four components. Context first - one paragraph on who this agent is and what it knows. Task second - one specific paragraph on what it does with what input. Constraints third - two or three things it must never do. Output format last - describe the result in enough detail that someone reading your brief could sketch it on paper. Post your brief in the chat when the trainer asks.

What to notice

The output format section is the hardest to write. That difficulty is telling you something - most people have not thought clearly enough about what they actually want the agent to produce.

06 Advanced

Edge Cases: Where Agents Break

Agents follow instructions literally. Which means they will do exactly what you said - even when what you said does not cover a situation you did not anticipate.

Edge cases are the requests your brief did not address. When they arrive, the agent guesses. And it will guess differently every time.

1

The empty input

What does your agent do when the input is blank or missing? If your brief does not say, the agent will decide - and not always correctly.

2

The wrong format input

What does your agent do when it receives a PDF instead of a Word doc, or a table instead of a paragraph? Cover the most likely variants explicitly.

3

The ambiguous request

What does your agent do when the request could mean two different things? Should it pick one? Ask for clarification? Default to the most common interpretation?

Key Insight

A brief that does not address edge cases is a brief that is waiting to fail. You do not need to cover every possibility. Cover the three most likely failure modes and your agent will handle 90 percent of situations correctly.

07 Leadership

The Leadership Layer

Here is the organizational version of this skill. When an agent brief lives only in your head, it leaves when you do. When it is written down and tested, it is a system asset.

The AI Officers who advance fastest are the ones who document their briefs well enough that someone else could deploy and maintain the agent without asking them a single question.

Key Insight

A well-written brief is not just an agent instruction. It is organizational knowledge made explicit. Write briefs the way you would write a process document - as if the person who reads it next is smart, capable, and has no context.
